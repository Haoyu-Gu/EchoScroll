# EchoScroll · 演示物料清单与样本对生产计划

> 现场答辩 / 演示**绝对不能现场生成**——MusicGen 即使在 A100 上推理一段也要 20-40s，
> 网络抖动、模型加载、CUDA OOM 任意一个出问题就翻车。
> 本文档列清楚需要**提前离线烤好**的全部样本对、文件命名、放置路径与播放时机。

---

## 0. 演示信心三档

| 档 | 含义 | 实现工作量 |
|---|---|---|
| 🥇 **完整版** | 8 画 × 6 变体 = 48 音频，全部是 GPU 服务器 MusicGen-small + LoRA 真生成 + M5 派生 | ~4 GPU 小时一次性烤 |
| 🥈 **Stable 版** | 8 画 × 4 变体 = 32 音频，只生成 default + 3 个 M5 变体 | ~2 GPU 小时 |
| 🥉 **Mock 兜底** | 当前 demo 内的 7 个 mock sine 音频，原样使用 | 已就绪（当前状态） |

> **强烈建议至少做 🥈 Stable 版**——MusicGen-small 真音色比 mock sine 听感差 100 倍，
> 答辩老师 30 秒就分辨得出。Mock 版只是 fallback，不该作主线。

## 1. 8 张画作 · 每张需要的 6 种音频样本

每张画对应一组完整的(画 + 元数据 + 多版本音频) 包。

### 样本对结构（每画都一样）

```
samples/<画作 ID>/
├── painting.jpg                  画作图像 (CC0)
├── metadata.json                 全部元数据（V-A / descriptors / RAG）
├── audio_01_default.wav          ★ 主版：MusicGen 直出，8s @ 32kHz mono int16
├── audio_02_slow.wav             M5 phase-vocoder ×0.5 BPM，约 16s
├── audio_03_fast.wav             M5 phase-vocoder ×1.5 BPM，约 5.3s

└── audio_06_baseline.wav         去掉 RAG 与 V-A 条件，仅文字 prompt 的对照版（评测用）
```

## 2. M5 编辑变体生产

只要有了 `audio_01_default.wav`（MusicGen 直出），M5 变体全部本地用 librosa 派生：

| 变体 | 算法 | 代码 |
|---|---|---|
| `audio_02_slow` | `librosa.effects.time_stretch(rate=0.5)` | M5 模块 `change_bpm(src=120, tgt=60)` |
| `audio_03_fast` | `librosa.effects.time_stretch(rate=1.5)` | M5 模块 `change_bpm(src=120, tgt=180)` |
| `audio_04_segment_edit` | 中段（35%-60%）替换为带白噪声的同长度片段 + 50ms cross-fade | M5 模块 `replace_segment(idx=2, replacement=...)` |

→ **不需要 GPU**，CPU 几秒搞定。

## 3. M7 哼唱配对（如果要展示哼唱交互）

需要预录 **8 段哼唱**，每段 3-5s，每张画一段，作为"我想让旋律走这样"的输入：

| 画 | 哼唱内容 |
|---|---|
| p1 云山图 | A 小调下行三度（A-G-E），含两个长音 |
| p2 林逋月夜 | C 大调三和弦上行 (C-E-G)，每音 0.6s |
| p3 群仙山图 | D 调五声上行音阶 (D-E-F#-A-B) |
| p4 溪山兰若图 | F 小调长音持续（一个 F 音，带渐强）|
| p5 仿郭忠恕山水 | 自由律动短音群（高、低、高、低）|
| p6 溪山无尽图 | G 调铿锵节奏型 (G-G-G-D-G) |
| p7 寒梅图 | E 小调下行二度 (E-D-C 三音) |
| p8 雪梅图 | C 大调高音点 (C5-E5-G5)，短促 |

录制要求：
- 22050 Hz, mono, WAV
- 用钢琴/合成器代替人声（更稳定 + 不涉及肖像权）
- 文件名：`samples/<id>/humming_input.wav`

M7 处理后的输出（pYIN F0 + DTW 转调）：
- 文件名：`samples/<id>/audio_05_humming_aligned.wav`

如果时间紧，**这部分可以省**——演示时口头说"系统也支持哼唱交互"+给一张 M7 的 figure 截图即可。

## 4. 基线对照音频（评测演示用）

按报告 §3.3.3 的盲评要求，至少准备 **2 种 baseline** 与 EchoScroll 对照：

| 系统 | prompt | 来源 |
|---|---|---|
| **A · EchoScroll** | 完整 (V-A + RAG + LoRA) | 上面的 `audio_01_default` |
| **B · text-only baseline** | 仅画作标题做 prompt，如 *"Chinese landscape painting soundtrack"* | MusicGen-small 直出，不接 V-A 不接 RAG |
| **C · no-RAG ablation** | 有 V-A 但去掉 RAG 上下文 | MusicGen-small 直出 |

每画都做 A/B/C 三组 → **8 × 3 = 24 个对照音频**。

放置：`samples/<id>/baselines/` 子目录。

> 这一批是**评测演示**用的——答辩时可以放 1-2 组（如 p2/p6）让评委盲听，
> 论证 EchoScroll 的优势不是凭空说的。

## 7. 物料生产工序（建议跑的顺序）

### 阶段 A · 服务器 GPU（一次性 ~4 小时）

```bash
# 在挂了 GPU 的服务器
cd echoscroll/M4_music_generator

# 1. 拉真 MusicGen-small（首次 2 GB）
python demo.py --real

# 2. 批量生成 24 个音频（8 画 × 3 系统）
python batch_generate.py \
    --paintings ../../samples/manifest.json \
    --systems A,B,C \
    --duration 8 \
    --out ../../samples/
```

> `batch_generate.py` 这个脚本现在还没写。要写就是个 for 循环，每画喂 prompt 给 `MusicGenWrapper.generate()`。

### 阶段 B · 本地 CPU（~30 分钟）

```bash
cd echoscroll/M5_editing_layer
python batch_edit.py \
    --samples ../../samples/ \
    --variants slow,fast,segment
```

→ 派生出每画的 audio_02 / audio_03 / audio_04。

### 阶段 C · 录制哼唱（可选，~1 小时）

```bash
cd echoscroll/M7_humming_interaction
python batch_humming.py \
    --samples ../../samples/
```

### 阶段 D · 接入网页（~30 分钟）

把 `samples/` 下的所有 audio 拷贝到 `echoscroll_demo/assets/audio/real/`，
然后改 `app.js` 里的 `PAINTINGS[*].audio` 字段指向真音频路径。

### 阶段 E · 录屏备份（~10 分钟）

QuickTime 录满流程一遍，存为 `samples/_backups/demo_full_4min.mp4`。

## 9. 现场容错预案

| 失败情况 | 应对 |
|---|---|
| 网络断 / Google Fonts 加载失败 | 字体退化到系统字体，整页功能不受影响。**不慌**。|
| 浏览器音频被禁（autoplay policy）| 先在 Hero 区**点击一次任意位置**激活 audioContext。这是 demo 一开始必须做的"开机动作"。|
| 某段音频文件丢失 | 网页继续运行，那段播放器静音。**演示主线不依赖单一文件**。|
| 整页打不开（CSS/JS 报错）| 切到 `samples/_backups/demo_full_4min.mp4` 全屏播放。|
| 投影分辨率怪 | 在浏览器设置 zoom level（Cmd+/-）即可全屏适配。|
| Concert Mode 跑飞 | ESC 退出。或 Cmd+R 刷页重来。|

      "artist": "Du Jin",
      "dynasty": "明 Ming",
      "image": "samples/p2_dujin_lin_bu/painting.jpg",
      "va": [-0.03, 0.11],
      "word": "tender",
      "descriptors": {

        "baseline_text_only": "Chinese landscape painting soundtrack",
        "ablation_no_rag": "Ming literati ... (without retrieved context)"
      },
      "audios": {
        "default":          "samples/p2_dujin_lin_bu/audio_01_default.wav",
        "slow":             "samples/p2_dujin_lin_bu/audio_02_slow.wav",
