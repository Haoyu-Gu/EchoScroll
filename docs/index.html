EchoScroll

Where Chinese Painting Listens to Itself.

让一幅中国画自己讲述它的音乐。EchoScroll 把视觉语义、艺术史语境与情感几何对齐到同一个潜在空间，再交给可控的 MusicGen 完成配乐生成。我们不再用"中国风"标签敷衍古画，而是让每一卷纸都得到与它气韵相称的声音。

Vision

让画自己听见

中国画的妙处在于多模态——视觉、题跋、印章与诗书互为表里。但今天的数字渠道把它压缩成一张静止缩略图，失去了与观者气息往返的渠道。

EchoScroll 通过 CLIP + Sentence-BERT 编码视觉与文本，对齐到 Russell 二维情感空间， 融合艺术史 RAG 检索的文化语境，再用 LoRA 微调的 MusicGen 生成与画作气韵相称的配乐—— 全过程可拖动、可重生成、可哼唱微调。

Research Context · §2 of Proposal

研究语境

三栏对照看清 EchoScroll 在文献与产业景观中的位置：现状有什么 / 缺什么 / 我们做什么。

State of the Field

现状

文本到音乐生成在 2023 之后高速演进：MusicGen / MusicLM / AudioGen / Stable Audio 等已可从一句 prompt 出可用配乐；情感条件化也有 RE-MAST 等 V-A 直接控制的工作。

  • MusicGen 系列覆盖中等 token-level 多条件生成
  • RE-MAST 实证连续 V-A 控制优于离散标签
  • Suno / Udio 商业闭源，质量好但不可控不可解释
  • image-to-music 工作多停在视觉风格迁移层
The Gap

差距

没有任何系统把"中国画的画派/题跋/构图/留白"作为一等的条件化信号；现有方案处理"中国风"基本停留在 prompt 里塞几个国乐乐器名的标签级敷衍。

  • 没有针对中国画的 V-A 训练数据
  • 没有可拖动可编辑的 V-A 用户接口
  • 艺术史 RAG 几乎无人做（仅 GalleryGPT 间接相关）
  • 评测维度缺"文化适配 cultural appropriateness"
Our Approach

EchoScroll 的方法

把"画意 → 情感几何 → 音乐参数"明确为一条可观察、可干预、可解释的三段管线，而不是黑盒一阶端到端；中国画的艺术史语境通过 RAG 注入。

  • 多模态 z + V-A 投影：两步可视、可拖动
  • BGE-M3 + FAISS 检索 1129 条策展文 chunk
  • LoRA 微调 MusicGen 走中国民乐音色
  • 评测含 cultural appropriateness 单独打分
Pipeline · §3.2 of Proposal

五步而成

壹

感知 · Perception

M1 · CLIP/ViT + BGE-M3

把画作图像、题跋诗文与元数据（朝代/画家/画派）分别编码，再融合成 768 维多模态向量 z。

贰

映射 · Projection

M2 · MLP → V-A

两层 MLP 把 z 投到 Russell 圆环 (v, a) ∈ [-1, 1]²，用 MSE + NT-Xent 联合监督。

叁

检索 · Retrieval

M3 · BGE-M3 + FAISS

从艺术史语料库（1,129 chunks）取回 top-k 文化语境，避免泛"中国风"。

肆

生成 · Generation

M4 · MusicGen + LoRA

V-A、口语指令、RAG 上下文共同条件化 MusicGen-small；LoRA 适配中国民乐。

伍

编辑 · Refine

M5/M6/M7 · Edit + Hum

段位替换、相位声码器变速、哼唱旋律对齐——把生成结果交还用户继续打磨。

回声画乐

EchoScroll · 回声

Where Chinese Painting Listens to Itself.

项目本旨

中国画的妙处在于多模态——视觉、题跋、印章、诗书互为表里。但今天的数字渠道把它压缩成一张静止缩略图，失去了与观者气息往返的渠道。EchoScroll 把"画 → 情感 → 音乐"明确为一条可观察、可干预、可解释的三段式 pipeline，用 V-A 二维情感几何作为人机共同语言。

为什么这件事值得做

从技术看，它把多模态语义、艺术史 RAG、可控音频生成三个研究脉络拧到一起；从文化看，它给"中国风" AI 配乐这一被劣化处理的应用场景提供一个有交互、有解释、有审计性的替代方案。

底层架构

9 个独立模块（M1 多模态编码 / M2 V-A / M3 RAG / M4 MusicGen+LoRA / M5 编辑 / M6 口语翻译 / M7 哼唱 / M8 前后端 / M9 评测）通过共用的 V-A 与音乐描述符 schema 串联；后端 FastAPI，前端 React + WaveSurfer。

数据层包括 Met / Cleveland 的 CC0 中国画 ~3K 张、EmoArt-130k 全集 V-A 标签、CCMusic 中国民乐 ~80h、自标 V-A pilot 2K 张。
